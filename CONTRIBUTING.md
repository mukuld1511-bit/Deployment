# Contributing Guidelines

Thank you for your interest in contributing to this project! To keep the project clean and maintainable, please follow these guidelines.

## How to Contribute

1. **Fork and Clone**: Fork the repository and clone it locally.
2. **Create a Branch**: Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make Your Changes**: Write clean, commented code and ensure tests pass.
4. **Commit**: Write clear, descriptive commit messages.
5. **Submit a Pull Request**: Push your branch to GitHub and open a Pull Request.

## Code Style

- Follow standard Java code formatting conventions.
- Keep components focused and modular.
- Add comments and documentation for public APIs and complex logic.
